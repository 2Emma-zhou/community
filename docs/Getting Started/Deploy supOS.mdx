## Installing supOS
### Linux
1. Make sure you have the following system and components ready.
- Ubuntu Server 24.04
- Docker

<table>
<thead>
<tr>
<th>Component</th>
<th>Version</th>
</tr>
</thead>
<tbody>
<tr>
<td>Docker Engine - Community</td>
<td>27.4.0</td>
</tr>
<tr>
<td>Docker Buildx</td>
<td>v0.19.2</td>
</tr>
<tr>
<td>Docker Compose</td>
<td>v2.31.0</td>
</tr>
<tr>
<td>Containerd</td>
<td>1.7.24</td>
</tr>
</tbody>
</table>

:::note
Above system and components have been tested. You can try on other versions if you are interested.
:::

2. Clone the project from Github.
```
git clone https://github.com/FREEZONEX/supOS-CE.git
```

3. Go to the folder **supOS-CE** inside the project.
```
cd supOS-CE
```
4. Enter the **.env** file and press the **i** key to edit configuration items as needed.
```
vi .env
```
<table>
  <tr>
    <th>Item</th>
    <th>Description</th>
  </tr>
  <tr>
    <td>VOLUMES_PATH</td>
    <td>The storage path for project data.</td>
  </tr>
  <tr>
    <td>ENTRANCE_DOMAIN</td>
    <td>supOS access address. Normally the IP or domain of the server where you install supOS.</td>
  </tr>
  <tr>
    <td>ENTRANCE_PORT</td>
    <td>supOS access port. Make sure the port is not occupied.</td>
  </tr>
  <tr>
    <td>LANGUAGE</td>
    <td>supOS display language. Chinese (zh-CN) and English (en-US) are supported.</td>
  </tr>
</table>

<img width={550} src="http://communityimage2.oss-cn-hangzhou.aliyuncs.com/53.png" />

:::tip
We have provided *install-docker* script to help you quickly set up the docker environment that is comptible with Ubuntu Server 24.04, and stored it under **supOS-CE/bin/install-deb**.
:::

5. Press **ESC** to stop editting, save the file and install supOS-CE.
```
:wq!
bash bin/startup.sh
```
6. Select whether to keep the entered entrance address and whether to use default setup, and then wait for the installation to complete.

<img width={550} src="http://communityimage2.oss-cn-hangzhou.aliyuncs.com/54.png" />

7. Access supOS on a browser and log in.

<img width={550} src="http://communityimage2.oss-cn-hangzhou.aliyuncs.com/55.png" />

### Windows

:::info
- Make sure you have installed the lastest **Docker Desktop** and **Git** on Windows 10 or 11.
- Docker must be started.
:::

1. Clone the project from Github in **Git Bash**.
```
git clone https://github.com/FREEZONEX/supOS-CE.git
```
2. Go to the folder **supOS-CE** inside the project.
```
cd supOS-CE
```
3. Edit the following attributes in file **.env**.
:::tip
Before editing .env file, execute `ipconfig` in Git Bash to get the IP of the server.

<img width={350} src="http://communityimage2.oss-cn-hangzhou.aliyuncs.com/144.png" />

:::
- **OS_PLATFORM_TYPE**: The operating system where supOS is installed. Change it to **windows**
- **VOLUMES_PATH**: The storage path for project data.
- **ENTRANCE_DOMAIN**/**ENTRANCE_PORT**: supOS access address. Normally the IP or domain of the server where you install supOS, and make sure the port is not occupied.

```
vi .env
```
:::note
**.env** file is hidden, directly enter the command to open it.
:::

<img width={550} src="http://communityimage2.oss-cn-hangzhou.aliyuncs.com/56.png" />

4. Save the file and start installing supOS.
```
:wq!
bash bin/startup.sh
```
5. Select the default configurations and wait for the installation to complete.

<img width={550} src="http://communityimage2.oss-cn-hangzhou.aliyuncs.com/57.png" />

6. Access supOS on a browser and log in.

## Uninstalling supOS
1. Access the path **supOS-CE/bin**.
2. Execute the script **uninstall.sh**.
```
bash uninstall.sh
```
:::info
- **clean-all.sh** will wipe out all project data.
- Images will not be deleted.
- If you want to install supOS again, make sure you have pulled all images with the newest versions (based on image tags).
:::
